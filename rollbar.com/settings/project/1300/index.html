Webhook

If you provide a URL here, we'll POST to it on each of the notification events (new items, reactivated items, 10th/100th/etc. occurrence). The payload is JSON and includes the event name, the related item, and the related instance.

Note that this URL will receive ALL such notification events, for all environments and levels.

Webhooks should return a 200 response code. If we receive a different response code, or can't connect, we'll retry up to 5 times at a 5-second interval before giving up.

Better documentation is coming soon.
